1. 使用img标签： 适合简单显示静态 SVG 图标。
    • 优点：简单易用。
    • 缺点：无法直接在 SVG 内部进行样式或脚本控制，不能动态更改 SVG 内容。
  1. 使用object标签： 适合需要交互和样式控制的 SVG。
    • 优点：支持动态交互和样式控制，可以嵌入外部资源。
    • 缺点：在某些旧版浏览器中的支持不如 img 标签，需要处理可能的安全性问题（如跨域访问）。
  1. 直接嵌入svg文件： 适合需要直接控制和修改 SVG 内容的情况。
    • 优点：完全控制 SVG 内容和样式，可以直接在 HTML 中修改和操作 SVG 元素。
    • 缺点：增加了 HTML 文件的体积，不易重用，需要复制粘贴相同的 SVG 代码。
  1. 使用CSS背景图： 适合用作装饰性背景图像的 SVG。
    • 优点：适合使用在背景或装饰元素中，可以通过 CSS 控制显示和布局。
    • 缺点：无法控制 SVG 内部样式和脚本，不支持交互操作。
  1. 使用iframe标签： 适合隔离复杂 SVG 文档或应用。
    • 优点：与 object 类似，可以加载外部资源，隔离了 SVG 文件的作用域，防止样式和脚本冲突。
    • 缺点：性能开销较大，每个 iframe 都会创建一个新的浏览上下文，不太适合小图标，更多用于嵌入复杂的文档或应用。
